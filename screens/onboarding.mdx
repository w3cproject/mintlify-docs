## Как получить

Получить данные для экранов **onboarding** можно с помощью `GET` запроса к API `/onboarding`.

<Card
  title="Get onboarding list"
  icon="link"
  href="/api-reference/onboarding/get-onboarding-list"
>
  Перейти к роуту и изучить основные поля
</Card>

<Note>
  Данный роут **НЕ** требует JWT авторизации для получения данных.
  
  Все ответы возвращаются с сортировкой по полю `queue` от меньшего к большему.
</Note>

## Список элементов onboarding

Метод `/onboarding` возвращает определенное кол-во элементов, которое может быть динамическим.
Если этих элементов 5 - отображаем 5 шагов на клиенте и т.д.

### Структура ответа

Метод возвращает массив объектов:
```json
{
  "id": "int",
  "heading": "string",
  "detail_desc": "string",
  "picture_url": "string",
  "queue": "int",
  "is_latest": "bool"
}
```

где:

`queue` - Целое число, которое определяет порядок отображения (очередность)
`is_latest` - Булевое значение, которое показывает, что элемент последний (для него другое отображение кнопок)
