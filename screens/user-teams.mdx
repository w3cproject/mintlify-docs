## Как получить список групп

Получить список групп можно с помощью `GET` запроса к API `/users/teams`.

<Card
  title="Get User teams"
  icon="link"
  href="/api-reference/users/get-user-teams"
>
  Перейти к роуту и изучить основные поля
</Card>

<Note>
  Данный роут **требует** JWT авторизацию для получения данных.
</Note>

### Структура ответа

Метод возвращает массив объектов групп с ключем `members` (массив юзеров)

```json
[
  {
    "id": "int",
    "name": "string",
    "members": [
      {
        "id": "string",
        "phone": "string",
        "first_name": "string",
        "last_name": "string",
        "avatar_url": "string"
      }
    ]
  }
]
```

## Как создать группу

Создать группу можно с помощью `POST` запроса к API `/users/teams`.

<Card
  title="Create user team"
  icon="link"
  href="/api-reference/users/create-user-team"
>
  Перейти к роуту и изучить основные поля
</Card>

<Note>
  Данный роут **требует** JWT авторизацию для получения данных.
</Note>

### Структура JSON Body

```json
{
  "name": "string",
  "members": ["string"]
}
```

где,

`members` - массив строк, состоящий из телефонов пользователей

### Структура ответа

Метод возвращает созданный объект группы с ключем `members` (массив юзеров)

```json
{
  "id": "int",
  "name": "string",
  "members": [
    {
      "id": "string",
      "phone": "string",
      "first_name": "string",
      "last_name": "string",
      "avatar_url": "string"
    }
  ]
}
```

Обновить группу можно с помощью `PUT` запроса к API `/users/teams/{id}` по `ID` группы.

<Card
  title="Update user single team"
  icon="link"

>
  Перейти к роуту и изучить основные поля
</Card>

<Note>

  Объекты для передачи в `body` и структура ответа совпадает с роутом создания группы (`POST /users/teams`)
